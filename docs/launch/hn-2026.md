## Body

I posted Chidori here in 2023 [1] as a "declarative framework for AI agents": a reactive graph, an embedded Deno + Starlark interpreter, time-travel debugging, evals. Three years and a near-total rewrite later, the goal is unchanged and almost everything else is different.

The biggest change is what an agent is. The graph is gone. Nobody thinks in reactive nodes, and agents get generated now anyway — models write clean TypeScript far better than any bespoke format. So an agent is one plain async TypeScript function:

  export async function agent(input, chidori) {
    const plan = await chidori.prompt("Plan: " + input.bug);
    const diff = await chidori.tool("repo.apply", { plan });
    await chidori.input("Ship it?", { type: "approval" });
    return chidori.prompt("Changelog:\n" + diff);
  }

That's a complete production agent with a tool call and a human approval gate. There is nothing else to learn. Everything arrives through that chidori object: prompts, typed tools written as ordinary TS files, MCP servers, human input and named signals, multi-turn conversations plus a built-in chat REPL, parallel branches across real OS threads, and a filesystem workspace scoped to the project directory. Providers are Anthropic, OpenAI, OpenRouter, and LiteLLM-compatible proxies, and chidori login signs into OpenRouter from the browser, so you can try it without an API key.

Here's what makes the plain function viable in production: the agent never touches the world directly. Every side effect goes through the runtime as a recorded host call, so a run is a journal. Kill the process mid-run and resume in a new process at the exact step. Pause for that approval and pick it up days later without holding anything open. Fork competing strategies across threads. Commit a recorded run to git and it's a regression test that runs in milliseconds. The function above contains no retry logic and no checkpointing code, because the runtime absorbs the production concerns.

To be precise about the replay claim: the first run is as non-deterministic as any LLM. What's guaranteed is that re-execution against the recorded journal is byte-identical, and a checkpoint is validated against the agent source it came from, so a stale journal fails loudly. That guarantee required owning the clock, randomness, and every I/O call the agent can observe, which is why agents run on Chidori's own JavaScript engine, built from scratch in Rust. oxc parser, bytecode compiler, stack VM, zero unsafe. It passes 99% of the Test262 tests it executes and skips about 7.5k tests for features it doesn't implement, with every skip listed. As proof the engine isn't a toy, unmodified React 18 renders on it, and an agent can iterate on a component against its own DOM tests, replaying its earlier model calls for free.

I should be honest that a project this size got built by one person directing coding agents: the engine, the Test262 grind, much of the runtime. The reason I could trust code I didn't type is the same loop I'm shipping. Every change gated by conformance tests, every run recorded and replayable. That's also the longer bet: if tools are just TypeScript and agents write good TypeScript, an agent can factor its own hard-won work into typed tools it reaches for later. I'm early on that loop, but it's what this architecture is for.

On the now-crowded field: Temporal replays journaled histories too, and has for a decade, but you split your code into workflows and activities and keep the workflow half deterministic by discipline; break the rule and replay diverges. LangGraph checkpoints between graph nodes. Vercel's eve records traces of agents that get a real shell. Chidori's difference is that there is no split and no discipline to maintain. Determinism is a property of the engine, so it can't be violated by user code. It also ships as one binary, no cluster, no Postgres, no Node, and chidori run --isolate puts untrusted code in its own child process behind seccomp, Landlock, and an empty network namespace (Seatbelt on macOS).

Honest cost: an interpreter with no JIT runs compute-heavy microbenchmarks 10-40x slower than Node or Bun, worse on tight numeric loops. The benchmark suite is in the repo. Agent workloads are dominated by model and I/O waits where this barely registers, and startup is about 10x faster than Node.
